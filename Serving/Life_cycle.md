## 문제 정의의 중요성
* 특정 현상 파악 => 그 현상의 있는 문제를 정의
* 문제를 명확하게 정의해야 한다
* 현상 파악 -> 문제 정의 -> 프로젝트 설계 -> Action -> 추가 원인 분석

#### 현상 파악
* 어떤 일이 발생
* 어려움은 무엇인가
* 해당 일을 해결하면 좋은 것은 무엇인가
* 어떤 가설을 만들어 볼 수 있을까
* 어떤 데이터가 있을까

#### 구체적 문제 정의
* 무엇을 해결하고 싶은가
* 무엇을 알고 싶은가

#### 프로젝트 설계
* 문제 정의
* 최적화할 Metric 설계
* 데이터 수집
